webpack

1、模块化开发中，我们会编写大量模块，如果不打包就上线，那么页面加载或交互时，将会发起大量请求。为了性能优化，需要使用webpack这样的打包器对模块进行打包整合，以减少请求数。流行的打包工具有 Gulp, Grunt, Webpack等工具

2、 webpack:一款模块加载器兼打包工具，它能把各种资源文件都作为模块来使用和处理,可以参考下图来理解

3、特性：

  • 以 commonJS 的形式来书写脚本

  • 开发方便且提高浏览器性能

  • 有很多模块加载器可用

4、模式

  • 生产模式，压缩代码

  • 开发模式：不压缩代码

npm

1、定义：NPM是随同 NodeJS一起安装的包管理工具，能解决NodeJS代码部署上的很多问题，常见的使用场景有以下几种：

  • 允许用户从NPM服务器下载别人编写的第三方包到本地使用。

  • 允许用户从NPM服务器下载并安装别人编写的命令行程序到本地使用。

  • 允许用户将自己编写的包或命令行程序上传到NPM服务器供别人使用。

2、组成部分： 命令行、网站、注册表

  • 网站 是开发者查找包（package）、设置参数以及管理 npm 使用体验的主要途径。

  • 注册表 是一个巨大的数据库，保存了每个包（package）的信息。

  • CLI 通过命令行或终端运行。开发者通过 CLI 与 npm 打交道。

webpack和npm关系

1、webpack是npm生态中的一个模块，我们可以通过全局安装webpack来使用webpack对项目进行打包

2、webpack的运行依赖于node的环境，没有node是不能打包的，但是webpack打包后的项目本身只是前端静态资源和后台没有关系，也就是说不依赖与node，只要有后台能力的都可以部署项目

3、npm是于Node社区中产生的，是nodejs的官方包管理工具，当你下载安装好node的时候，npm cli 就自动安装好了

4、正是因为npm的包管理，使得项目可以模块化的开发，而模块化的开发带来的这些改进确实大大的提高了我们的开发效率，但是利用它们开发的文件往往需要进行额外的处理才能让浏览器识别，而手动处理又是非常繁琐的，这就是webpack工具存在的意义
