1、什么是双向绑定

  • 双向绑定：数据和视图之间，一方改变就会引起另一个改变，二者的数据同步变化

  • 实现思路: 二者间某一方的数据发生了变化，监测到这个变化后，通知其他的和该数据有联系的部分，让他们也跟着一起变化。

  • 实现方法：

    • 原理：vue中有一个自定义的方法：Object.defineProperty()可以拦截到变化数据，并且把data对象中数据用get/set方法保存下来，之后通知相应视图发生改变

    • 实现：实现数据的双向绑定，首先要对数据进行劫持监听，所以我们需要设置一个监听器Observer，用来监听所有属性。如果属性发上变化了，就需要告诉订阅者Watcher看是否需要更新。因为订阅者是有很多个，所以我们需要有一个消息订阅器Dep来专门收集这些订阅者，然后在监听器Observer和订阅者Watcher之间进行统一管理的。

3、跨域

  • 是什么：跨域即为跨域名；可以粗略理解为从一个域名（页面）跨到另一个域名（页面）时，请求会被浏览器拦截。

    • 域名 = 协议:// 主机地址：端口号

    • 因此只要两个页面的 协议、主机地址、端口号 三者中有任何一个不同，则它们属于两个不同的域名，它们通信间即为跨域

  • 名词提出：浏览器为了正常、安全使用，具有一个安全功能：同源策略；同源策略会阻止一个域的javascript脚本和另外一个域的内容进行交互

  • 方法:

    • document.domain：非同源网页的 Cookie问题，只适合主域名相同，但子域名不同的iframe跨域。

    • window.postMessage()：跨文档通信 API

    • JSONP：是服务器与客户端跨源通信的常用方法，但只支持get请求，不支持post请求。

    • CORS：是跨域资源分享（Cross-Origin Resource Sharing）的缩写。它是 W3C 标准，属于跨源 AJAX 请求的根本解决方法。详解可移步

4、代理

  • 顾名思义即为在请求方向被请求方发送请求时，让一个中间层域名代理这个请求或说代替请求方向被请求方发送请求，从而解决两方不在同一地域的问题

  • 实现：客户端先发送请求到代理中间层，之后由中间层向服务器发送请求；返回结果也是一样，先到中间层，再到客户端。

  • 正向代理、反向代理

    • 区别：代理是为谁服务的。

    • 正向代理：客户端请求代理帮忙 + 代理向服务端发送请求

    • 反向代理：客户端向代理发送请求 + 代理替服务器处理请求
