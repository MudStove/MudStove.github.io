3、跨域

  • 是什么：跨域即为跨域名；可以粗略理解为从一个域名（页面）跨到另一个域名（页面）时，请求会被浏览器拦截。

    • 域名 = 协议:// 主机地址：端口号

    • 因此只要两个页面的 协议、主机地址、端口号 三者中有任何一个不同，则它们属于两个不同的域名，它们通信间即为跨域

  • 名词提出：浏览器为了正常、安全使用，具有一个安全功能：同源策略；同源策略会阻止一个域的javascript脚本和另外一个域的内容进行交互

  • 方法:

    • document.domain：非同源网页的 Cookie问题，只适合主域名相同，但子域名不同的iframe跨域。

    • window.postMessage()：跨文档通信 API

    • JSONP：是服务器与客户端跨源通信的常用方法，但只支持get请求，不支持post请求。

    • CORS：是跨域资源分享（Cross-Origin Resource Sharing）的缩写。它是 W3C 标准，属于跨源 AJAX 请求的根本解决方法。详解可移步此处

4、代理

  • 顾名思义即为在请求方向被请求方发送请求时，让一个中间层域名代理这个请求或说代替请求方向被请求方发送请求，从而解决两方不在同一地域的问题

  • 实现：客户端先发送请求到代理中间层，之后由中间层向服务器发送请求；返回结果也是一样，先到中间层，再到客户端。

  • 正向代理、反向代理

    • 区别：代理是为谁服务的。

    • 正向代理：客户端请求代理帮忙 + 代理向服务端发送请求

    • 反向代理：客户端向代理发送请求 + 代理替服务器处理请求
