1、原因：

  • 各类浏览器的内核不同，导致它们解析起来略有差异，而主要内核：webkit、Gecko、Trident(IE内核)、Blink

  • webkit（Chrome）、Gecko（FireFox）、Trident(IE)、Blink

2、常见的兼容性问题及解决：

一：js【通过对内核的 if/else 来决定每个样式的显示效果】

1、const: Firefox下,可以使用const关键字或var关键字来定义常量;IE下,只能使用var关键字来定义常量

2、对象宽高赋值问题：FireFox中类似obj.style.height = imgObj.height的语句无效

二:CSS 【如下】

1、图片默认有间距 ==> 用float对img进行布局

2、边距重叠问题：当两个兄弟元素都设置了margin 边距时，margin 将选其中较大的值 ==> 给它们设立一个父元素，父元素的样式设置 overflow:hidden；

3、双边距问题： IE6中一个块设置了float , 同时又设置margin , 就会出现边距问题 ==> display:inline;
