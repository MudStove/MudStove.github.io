进程的基本内容

1、并发和并行

  • 并发：1个CPU轮流执行多个程序，因为时间间隔短，看起来两个程序都在执行，可实际上每个时刻只有一个程序在执行

  • 并行：多个CPU同时执行多个程序，二者是一对一关系，每个时刻都有多个程序在执行

2、进程的定义

  • 进程是可以并发执行的计算机部分

  • 是一个可以独立调度的活动

  • 是一个抽象实体，是一系列逐一执行的操作

  • 是程序在处理机上执行的活动

3、进程和程序区别

  • 进程是动态的，而程序是静态的

  • 进程具有并发的特性（独立性 + 异步性），而程序没有

  • 进程是竞争计算机系统资源的基本单位，从而其并发性受到系统自己的制约

  • 不同的进程可包含同一程序

4、进程控制狂（PCB）= 进程的描述信息 + 控制信息 + 资源信息

5、进程的5个基本状态： 初始状态、就绪状态、执行状态、等待状态、终止状态

  • 就绪状态–>执行状态:通过调度； 执行状态–>就绪状态: 因为时间片到了

  • 执行状态–>等待状态:等待某个事件发生而休眠； 等待状态–>就绪状态:等待某个事件发生而唤醒； 执行状态–>终止状态：完成

6、进程控制：

  • 定义：系统使用一些具有特定功能的程序段来创建、撤销进程以及完成进程各状态间的转换，从而达到多进程高效并发执行、协调实现资源共享的目的。

  • 用于进程控制的程序段 ==> 原语

7、进程互斥

  • 定义：不允许两个以上并发进程同时进入临界区，实现资源共享

8、进程同步

  • 定义： 一组并发进程因直接制约而互相发送消息，导致互相合作、等待，各进程按照一定顺序执行的过程称之为进程间同步

9、进程间通信

  • 定义： 顾名思义即为进程间互相传递信息

  • 通信方式：

    • 主从式

    • 会话式

    • 消息或邮箱机制

    • 共享存储区方式

10、死锁问题

  • 定义：指各并发进程互相等待对方所拥有的资源，而这些进程在未得到对方的资源前不会释放自己的资源，造成的大家都想得到却又都得不到，都不能继续执行的状况。

  • 起因：并发进程竞争资源

    • 根本原因： 系统所提供的资源个数少于并发进程所要求的该类资源数

  • 产生死锁条件

    • 互斥条件：资源不能同时被两个或两个以上并发进程所拥有

    • 不剥夺条件： 进程获得资源后不能被其他进程强行剥夺，只能等他自行释放资源

    • 部分分配：每次活动只申请得到部分所需资源，在等待申请的资源时，继续占有已得资源

    • 环路条件：进程需求资源构成一种环形链路，即链中每一个进程占有的资源是下一个进程所需要的。

  • 消除方法

    • 预防

      • 打破资源的互斥条件和不剥夺条件，例如允许资源同时访问等

      • 打破资源的部分分配条件

      • 通过设定优先级打破环路条件

    • 避免

      • 把进程分为很多步，每步所需资源是固定的，且一步内进程保持的资源数是不变的

    • 检测与恢复

      • 检测进程组是否构成资源的请求为环形

11、线程概要：

  • 基本概念

    • 产生原因：进程创建和切换的次数越多，开销就越大，从而服务器可以处理和支持的用户访问请求就越少。所以为了减少开销，提高效率节省资源，人们引入线程概念。

    • 定义：

      • 是进程的一部分，又称为轻权进程或轻量级进程

      • 是CPU调度的一个基本单位

  • 进程与线程区别

    • 它们都是处理机调度的基本单位，但线程改变意味着CPU执行过程的改变，进程所拥有的资源并没有发生变化

    • 进程是资源分配的基本单位，线程是进程一部分，多个线程一起分享该进程的所有资源

    • 进程有完整的虚地址，而线程没有

    • 进程和线程类似于一个活动和活动步骤的关系

  • 线程分类： 用户级线程、内核级线程

  • 线程的基本状态：执行、就绪、阻塞，面对基本状态改变产生的操作如下

    • 派生：进程/线程派生一个线程，到达就绪状态

    • 调度：就绪状态–>执行状态

    • 阻塞：执行状态–>阻塞状态

    • 激活：阻塞状态–>就绪状态

    • 结束：执行状态–>结束状态
