进程管理

文章涉及进程的一些基本信息与指令

基本概念

1、是什么

  • 简单来说，进程就是运行着的程序，在操作系统中有时也叫做任务

  • 官方定义：进程是程序在某个数据集合上的运行活动，是系统进行资源分配和调度的基本单位

2、类型：

  • 交互进程：既可以在前台运行，也可以在后台运行的进程，也是由某种shell程序启动的进程，比如一个命令

  • 批处理进程：是一个进程系列，和终端没有关系，例如执行一个脚本程序

  • 监控进程：也叫守护进程，在Linux系统启动时运行在后台的进程

3、进程和程序区别

  • 进程是程序处理数据的过程、而程序只是一组指令的有序集合

  • 进程是动态、异步、并发和独立的；而程序是静态的代码，不具备这些特性

  • 进程和程序是多对一的关系

进程的启动

1、手动启动

  • 前台启动 ：终端shell程序创建进程

  • 后台启动： 命令后加上 &

  • 前后台切换：

    • 前–>后: bg [工作编号]

    • 后–>前：fg [工作编号]

    PS: […] 为可选项

2、调度启动

  • 定义：通过事先的调度安排，让系统任务在指定的时间、场合运行，以满足系统自启动某些进程来完成工作

  • 命令

    • at

      • 作用： 用于在特定的时刻执行任务

      • 用法：at [-v] [-q queue] [-f file] [-mldbv] time

    • batch

      • 作用：在系统负载达到某一条件时自动执行命令

      • batch [-v] [-q queue] [-f file] [-mv] [time]

    • crontab

      • 作用：配置定时器，告诉提供定时器功能的cron守护进程让用户定时执行命令或列表，并维护每个用户的crontab文件

      • 用法：crontab [-u user] file [-e -l -r]

进程查看

1、ps命令

  • 用于查询进程的详细情况

  • 命令格式： ps [options]

2、top命令

  • 每隔一段时间，周期性的报告、实时的显示大多数以使用CPU为主的进程的状态信息

  • 命令格式： top [options]

进程调度

1、kill命令

  • 终止前台进程 <== 控制台组合键中断（ctrl+c/break）

  • 僵死的进程或后台进程 <== kill命令终止 或 top命令的k功能键

  • 工作原理: kill命令向一个或多个进程发送指定的信号，若没有明确的指定发送的信号，则默认发送的是 sigterm 信号（15号信号），而 sigterm 信号将终止所有不能捕获该信号的进程

    PS: 另一常用的信号： sigkill 信号（9号信号）

  • 命令格式： kill [-s signal / -p] [-a] [–] pid 或 kill -l [signal]

    • -l [signal]:列出kill命令使用的全部信号名称和编号

    • -s signal ： 指定要发送的信号

2、nice命令/renice命令

  • 作用：改变进程的优先级数值。高数值的进程能优先使用CPU 数值范围 -20~19

  • nice命令:

    • 用于以某一优先级启动一个进程

    • 命令格式：nice [option]

  • renice命令:

    • 用于改变一个正在运行的进程的优先级数值

    • 命令格式：renice 改变的优先级数值 pid(进程编号)
