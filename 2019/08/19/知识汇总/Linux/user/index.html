用户管理

2019-08-19
知识汇总
  • Linux
本文总阅读量次

本篇粗略介绍内容包括用户登录和账户管理

用户登录

1、终端登录/本地登录

  • Linux提供了6个虚拟控制台终端，每个终端都可登录，可使用命令 ctrl + alt + Fx切换【Fx = ‘F1~F6’】

2、远程登录：利用客户端登录到浏览器

  • 主要命令：telnet 、 rsh 、 rlogin 、 ssh

  • ssh客户端命令：

    • 由于rsh、rlogin 的安全性 ，ssh 替代了它们，因为 ssh 可以在不安全网络上两台非信任、不可靠的计算机间提供安全的加密通信

    • 通信时向远程计算机证明自己的身份： ssh -l login_name hostname/user@hostname command

      • login_name：登录的注册用户名

      • hostname/user@hostname: 远程计算机的名称、IP地址或域名

      • command：需要远程计算机shell执行的命令

用户账户管理

1、用户管理

  • 添加用户1： # useradd [-d home_dir] [-s sehll] 用户名

    • [-d home_dir] [-s sehll] 均为可选项，用于设置工作主目录，二者都不选时，默认工作主目录为： /home/用户名

  • 添加用户2：adduser + enter键【根据提示输入相应信息】

  • 用户信息保存

    • useradd命令建立的用户信息保存在 /etc/passwd 文件中

    • 加密的用户信息保存在 /etc/shadow 文件中

    • 系统在建立用户工作目录信息时，会初始化配置文件到该目录中【eg：组群】

    • 用户组相关信息保存在 /etc/group 文件中

    • etc: 配置文件目录

  • 添加用户3：用文本编辑器直接打开并修改 /etc/passwd 文件

    • 使用该方法不会自动创建用户工作目录，且不会修改shadow、group文件

    • passwd文件中信息格式： 用户名：密码：用户ID：组ID：用户全名：家目录/工作目录：创建用户时用户使用的shell

    PS: 文件每行有7个域，用 ：分隔开

    eg: yzy: xxxx: 0: 0: root: /root: /bin/bash

2、密码管理

  • 设置密码： passwd -k/d/u/l 用户名 【之后系统会提示输入密码】

    • -k:只有在原口令密码过期后才能修改

    • -d:删除口令密码

    • -u: 解开锁住的账户

    • -l:锁住账户

3、成批处理

  • 成批添加用户

    • 首先使用文本编辑器编写一个passwd中数据格式的txt新文件

    • 其次使用命令: newusers 文件名.txt ==> 读取文件中信息对，并更新passwd等文件中信息

  • 成批修改密码

    • newusers命令添加用户时还没有指定口令域，需要重新设置口令用户才能登录

    • 首先使用文本编辑器编写一个txt新文件，文件中为需要修改的用户名和密码，格式为： 用户名：密码

    • 其次使用命令: chpasswd < 文件名.txt

4、修改用户属性

  • 直接修改passwd文件

  • 指令 usermod 新属性 用户名

    • 新属性：格式类似于passwd文件中信息

    • 用户名：需要修改的用户名

5、删除用户

  • 直接删除文件中信息

  • 使用指令 userdel -r 用户名 【-r：可选项，指同时删除该用户目录下所有子目录和文件】

组群管理

1、添加组群

  • 修改文件添加

    • 直接打开 /etc/group 和etc/gshadow 文件修改

    • group格式： 组名：组群口令密码：组ID： 附加该组群的用户

  • 终端命令添加

    • groupadd [-o/r/f/g gid] 组名

2、添加组群口令密码

  • gpasswd 组名

3、修改用户组群属性

  • 修改文件: group和gshadow文件中信息

  • 命令行修改：groupmod [-g gid] [-o] [-n 新组名] 组名

4、删除组群

  • 修改文件删除

    • 同时删除group和gshadow文件中信息

  • 终端命令删除

    • groupdel 组名

5、每个目录文件中都有所有者权限、组权限、其他人权限；权限 = 读权限(r) + 写权限(w) + 执行权限(x)

  • ls 目录/文件 // 可查看目录/文件的权限

    • — — — ==> 体现是目录/文件 所有者的读写执行权限 组的读写执行权限 其他人的读写执行权限
