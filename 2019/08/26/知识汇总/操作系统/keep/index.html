存储管理

2019-08-26
知识汇总
  • 操作系统
本文总阅读量次

1、存储管理功能

  • 虚拟存储器

  • 地址变换

  • 控制内外存储数据传输

  • 内存的分配与回收

  • 内存信息的共享与保护

    • 保护方法： 硬件法、软件法、软硬件结合法

    • 上下界保护法：是一种硬件保护法

    • 保护键法

    • 界限寄存器和CPU用户态或核心态相结合方法

2、分区存储管理

  • 原理：给每一个内存中进程划分一块适当大小的存储区，用以连续存储各进程的程序与数据，使各程序能并发执行

  • 分类： 固定分区法 、 动态分区法

    • 动态分区： 在作业开始前不进行分区，其大小根据进程对内存的需求进行动态改变

    • 从可用表、自由链中寻找空闲区方法：

      • 最先适应算法：从搜索起始地址开始查找，若发现了大于或等于所需内存长度的分区则停止搜索

        • 搜索速度最快、回收过程最佳，仅可能的利用了低地址空间

      • 最佳适应算法：空闲区从小到大排序，从表头开始查找，若发现了可以满足需求的空闲区则停止搜索

        • 能找到最佳的空闲区，但却不一定能提高空间利用率

      • 最坏适应算法：空闲区从大到小排序，从表头开始检查，若可以满足需求的空闲区则停止搜索，否则失败

        • 为了不留碎片空闲区

  • 优点：

    • 实现了多作业或进程发共享，有助于多道程序设计，从而提高了资源利用率

    • 硬件支持少，算法简单易实现

  • 缺点：

    • 碎片问题，使得内存利用率还有待提高

    • 作业或进程大小受分区大小限制

    • 无法实现各分区间信息共享

3、覆盖与交换技术

  • 覆盖技术：同一程序内，把程序划分为若干功能独立的程序段，把不会同时执行的程序段放在一个分区内（一般是外存），当前面程序执行完后，把后续需要的程序段调入内存覆盖前面的已完成的程序段

  • 交换技术：不同程序间，把暂时不需要的程序数据保存在外存交换区，需要用到时，从外存交换区调取所需程序到内存中，实现内存的扩充

4、页式管理

  • 基本原理：把进程的虚拟空间划分为若干长度相等的页、内存空间也按照页的大小划分为片或页面

  • 页 = 页号P + 页内地址W ；每个页长大约为1K~4K

  • 静态页式管理

    • 需要为要求内存的进程或作业分配足够的页面，分配工作依赖三个表：页表、请求表、存储页面表

  • 动态页式管理

    • 抖动现象：页面调度非常频繁，以致于大多数时间花费在内外存的来回调入调出上，这一活动产生的现象叫抖动现象

    • 置换方法：

      • 随机淘汰算法：无法确认哪些页被访问的概率较低时，随机置换某一页

      • 先进先出算法（FIFO）：淘汰在内存中驻留时间最长的页

        • 内存利用率不高，假定CPU按线性顺序访问地址空间

        • 陷阱现象： 又称为Belady现象，即正常情况下却也次数和分配页面数成反比例增长，但是有时分配 页面数越多，缺页次数也随之增加

      • 轮转法：循环的方式置换出区内可以被换出的页，不考虑该区被换进的时长

        • 内存利用率不高，假定CPU按线性顺序访问地址空间

      • 最近最久未使用置换

      • 理想型淘汰算法

  • 优点

    • 由于不需要作业或程序在内存中连续存放，因此有效的解决了碎片问题

    • 提高了主存的利用率，有利于组织多道程序执行

  • 缺点

    • 需要程序的硬件支持

    • 增大了系统开销，例如缺页中断处理等

    • 页内利用率不够高

    • 请求算法的选择非常重要

5、段式管理

  • 基本思想：把程序按照内容或过程关系分成段，每段有自己的名字，对应一个二维虚拟空间

  • 优点：

    • 也是内外存统一管理，但是段式管理每次交换的都是有意义的信息

    • 段长可根据需要动态增长

    • 便于对有完整逻辑功能的信息进行共享

    • 便于实现动态链接

  • 缺点：

    • 需要硬件支持

    • 段长受到内存可用区大小的限制

    • 淘汰算法的选择也需要谨慎

6、段页式管理

  • 结合了段式管理与页式管理的优点

  • 增加了复杂性和开销

  • 一般只用于大型机系统
