OSI之数据链路层

2019-07-21
知识汇总
  • 计算机网络
本文总阅读量次

内容

1、数据链路层属于计算机网络的低层，该层使用的信道有两种：

  • 点对点信道: 一对一的点对点通信

  • 广播信道：一对多的广播通信

2、通信时的步骤

  • 把网络层交下来的IP数据报添加首部和尾部封装成帧

  • 把封装好的帧发送到目标节点的数据链路层

  • 目标节点的数据链路层判断收到的帧是否有差错，有则丢弃此帧，无则提取出IP数据报并上交给网络层

3、功能：

  • 封装成帧：把IP数据包转化成MAC帧

    • 帧为数据链路层的数据传送单元
    • 帧 = 帧首部（以“SOH”开头） + IP数据报（即“分组”） + 帧尾部（以“EOT”结尾）
    • “SOH”、“EOT”分别为帧开始符和帧结束符，用于确定帧的界限，即帧定界。
    • 每一种路由协议都会规定所能传输的帧的最大传送单元MTU ==> (IP数据包 <= MTU)

  • 透明传输: 让帧中除帧结束符“EOT”外的EOT像透明的字符一样传输，不影响正常传输

    • 解决方法: 在数据中的“SOH”、“EOT”前加上转义字符“ESC”。该方法被称为“字节填充或字符填充”

  • 差错检测

    • 比特差错：0、1在传输过程中出错在，转变成1、0

    • 循环冗余检验法（CRC）

  • 链路管理

  • 流量控制

  • 寻址

  • 将数据和控制信息区分开

4、主要协议：

  • PPP协议

  • CSMA/CD协议
