OSI之网络层

本文涉及网络层一些基本概念、IP等协议工作原理等

网络层基本概念

1、网络层提供的两种服务

  • 虚电路服务：在两台计算机通信前，要先建立连接(虚电路VC)

  • 数据报服务: 网络层向上提供提供的简单灵活、无连接、尽最大努力交付的服务

  • 网络层不提供服务质量的承诺

  • 虚电路服务和数据报服务区别

    对比的方面 虚电路服务 数据报服务
    思路 可靠通信由网络来保证 可靠通信由用户主机来保证
    是否要建立连接 需要 不需要
    终点地址 短的虚电路号 终点的完整地址
    分组的转发 同一虚电路的分组由相同的路由转发 每个分组选择独立路由进行转发
    出现故障时 不能工作 出故障节点可能丢失分组，一些路由可能发生变化
    到达顺序 和发送顺序相同 不一定和和发送顺序相同
    差错处理和流量控制 可由网络负责，也可由用户主机负责 用户主机负责

2、网络互连的中间设备

  • 物理层：转发器

  • 数据链路层： 网桥、桥接器

  • 网络层： 路由器

  • 网络层以上： 网关

3、网络层提供主机间的逻辑通信, 而传输层提供进程间的逻辑通信

网际协议IP

1、IP地址定义：

  • 就是给每一台主机或路由器的每一个接口分配的一个在全世界范围内唯一的32位标识符； 由互联网名字和数字分配机构ICANN分配。

2、IP地址编制历程：

  • 分类的IP地址

  • 子网的划分

  • 构成超网

3、IP地址/逻辑地址和硬件地址/物理地址

  • IP数据报 = TCP报文 + IP首部（含IP地址）

  • MAC帧 = 帧首部（含硬件地址） + IP数据报 + 帧尾部

4、两级IP地址

  • IP地址 = 网络号 + 主机号

    类别 网络号位数 主机号位数 最大网络数 地址范围
    A类地址 “0” + 7位 24位 2^7-2 1.0.0.0 ~ 127.255.255.255
    B类地址 “10” + 14位 16位 2^14-1 128.0.0.0 ~ 191.255.255.255
    C类地址 “110” + 21位 8位 2^21-1 192.0.0.0 ~ 223.255.255.255
    D类地址 “1110” + 多播地址
    E类地址 “1111” + 保留为今后所用

    PS1: 主机号位数 ==> 最大主机数 = 2^主机号位数

    PS2: 回环地址 ==> 127.0.0.1

5、划分子网：

  • 是什么： 两级IP地址 –> 三级IP地址

  • 为什么：

    • 两级IP地址的空间利用率低

    • 两级IP地址不够灵活：一个单位在开通新的网络，而在申请到之前，新增加的网络不能工作

    • 给每一个物理网络分配网络号会让路由表变得太大，从而损坏网络性能

  • 怎么做：

    • 借用部分主机号充当子网号

    • IP地址 = 网络号 + 子网号 + 主机号

  • 子网掩码

    • 作用：判断是否有划分子网以及确定子网是什么

    • 两级IP地址 的 子网掩码 = 网络号全置 1 + 主机号 全置 1

    • 三级IP地址 的 子网掩码 = 网络号和子网号全置 1 + 主机号 全置 1

    • 子网的网络地址 = 子网掩码 & IP地址

    • 子网数 = 2 ^ n - 2 （n为子网号的位数）

地址解析协议ARP

1、IP地址 （by ARP） –> 物理地址 （by RARP） –> IP地址

2、路由器：

  • 作用：缓存、查表、转发

  • 2个端口，分别连接2个网络

  • （二级IP地址）路由表 = 目的主机所在网络 + 下一跳地址

  • （三级IP地址）路由表 = 目的主机所在网络 + 子网掩码 + 下一跳地址

    PS: 通过 目的网络 & 子网掩码 ==> 目的网络地址 ==> 与 目的主机所在网络对比 ==> 得到正确的目的网络地址

网际控制报文协议ICMP

  • 作用： 能更有效的转发IP数据报

  • ICMP报文：装在IP数据报的数据部分

    ICMP报文种类 类型的值 ICMP报文的类型
    差错报告报文 3 终点不可达
    差错报告报文 5 改变路由（重定向）
    差错报告报文 11 时间超时
    差错报告报文 12 参数问题
    询问报文 8/0 回送请求或回答
    询问报文 13/14 时间戳请求或回答

    PS1: 回送请求或回答–>主机或路由器向一个特定的目的主机发出询问

    ps2: 时间戳请求或回答–>主机或路由器向一个特定的目的主机询问当前日期和时间

路由选择协议

1、静态路由选择、动态路由选择

  • 静态路由选择： 也叫非自适应路由选择策略 《==》 不能适应网络状态的变化，简单，开销小，适合较简单的小型网络

  • 动态路由选择： 也叫自适应路由选择策略 《==》 能适应网络状态的变化，复杂，开销大，适合较复杂的大型网络

2、自治系统 （AS）：互联网分割成许多小的网络，每个小网络便是一个自治系统

3、内部网关协议 （IGP）

  • 定义： 一个自治系统内使用的路由选择协议

  • RIP: 路由信息协议

    • 定义：是一种分布式的基于距离向量的路由选择协议（每经过一个路由器，距离++）

    • 距离也称跳数，if （跳数 = 16）==> 不可达

    • 工作原理：选择一条跳数最少（所要经过的路由器最少）的路径传输

    • 特点：

      • 仅和相邻路由交换信息

      • 交换的信息为主自己的所有信息，即交换现在的路由表

      • 固定时间间隔交换

      • 好消息传的快，坏消息传得慢

      • 实现简单，开销小

  • OSPF：开放最短路径优先

    • 使用分布式的链路状态协议

      • 采用洪泛法，向本AS中所有路由发送信息

      • 发送的信息是与之相邻的所有路由器的链路状态

      • 发送的触发条件为链路状态改变

    • 分组类型

      • 类型1：问候分组（Hello） ==> 用来和相邻路由建立连接

      • 类型2：数据库描述分组（DD） ==> 把自己链路状态数据库中所有链路状态项目的详细信息给邻站

      • 类型3：链路状态请求分组（LSR） ==> 向对方请求发送链路状态的详细信息

      • 类型4：链路状态更新分组（LSU） ==> 用洪泛法对全网更新链路状态

      • 类型5：链路状态确认分组（LSA） ==> 对链路状态更新的确认

3、外部网关协议 （EGP）

  • 定义： 多个自治系统内进行信息传递时使用的路由选择协议

  • 边界网关协议 （BGP）

    • 寻找一条能到达目的网络的较好的路由，而非最佳路由

    • 采用路径向量路由选择协议

    • 报文：

      • OPEN（打开）报文

      • UPDATE（更新）报文

      • KEEPALIVE（保活）报文

      • NOTIFICATION（通知）报文
