OSI之应用层

2019-07-28

  • 计算机网络
本文总阅读量次

应用层协议解决多个应用进程间的通信和协同工作；本文涉及DNS、万维网和HTTP协议等

1、应用层协议是基于客户服务器方式，客户和服务器均为应用进程；客户是服务请求方，服务器是服务提供方。

2、域名系统DNS

  • 作用:能够用于把主机名转化成IP地址

  • 域名结构 = …级域名.三级域名.二级域名.顶级域名

    PS1: 域名的标号由数字和英文字母组成，不区分大小写

    PS2：每一标号不超过63个字符，完整域名不超过255个字符

    PS3: 每一级域名即为一个标号

3、域名服务器：

  • 定义： 每一级域名都有一个域名服务器，一个服务器所负责管辖的范围就叫做“区”，每一个区设置相应的权限域名服务器

  • 根域名服务器

  • 顶级域名服务器

  • 权限域名服务器

  • 本地域名服务器

5、万维网

  • 定义： 是一个大规模的联机式的信息储藏所，简称Web；也是一个分布式的超媒体系统，是超文本系统的扩充

  • 客户向服务器程序发送请求，服务器程序返回客户所需要的万维网文档即网页。

  • UPL： 统一资源定位符；HTML:超文本标记语言

    • URL = 协议://主机:端口/主页路径 （端口可省略）

  • HTTP: 超文本传送协议，是面向事务、无连接、无状态的

    • 三次握手：建立连接阶段

      • 第一次：客户端向服务器发送请求报文段，其首部中：SYN=1,seq=x，并进入SYN_SEND状态，等待服务器确认。（SYN：同步序列编号）【客户端：“服务器~ 我想要建立连接”】

      • 第二次：服务器收到并确认,若同意请求，则发送确认报文段，首部中SYN=1,ACK=1,ack=x+1,seq=y 【服务器：“可以”】

      • 第三次：客户端收到确认报文之后，通知上层应用进程连接已建立，并向服务器发出确认报文，其首部：ACK=1,ack=y+1。当服务器收到客户端的确认报文之后，也通知其上层应用进程连接已建立。 【客户端：“已告诉应用层连接建立了”】【服务器：“我也已经告诉应用层连接建立了”】

    • 四次挥手：连接释放阶段

      • 第一次：数据传输结束以后，客户端的应用进程发出连接释放报文段，并停止发送数据，其首部：FIN=1,seq=u 【客户端：“我的数据传完了”】

      • 第二次：服务器端收到连接释放报文段之后，发出确认报文，其首部：ack=u+1,seq=v。此时本次连接就进入了半关闭状态，客户端不再向服务器发送数据。而服务器端仍会继续发送 【服务器：“好的，那我关闭你向我传递数据的连接啦，不过我还没传完，客户端你先别关闭我向你传递数据的连接噢”】

      • 第三次：若服务器已经没有要向客户端发送的数据，其应用进程就通知服务器释放TCP连接。这个阶段服务器所发出的最后一个报文的首部应为：FIN=1,ACK=1,seq=w,ack=u+1 【服务器：“客户端我传完啦，你可以关闭连接了”】

      • 第四次：客户端收到连接释放报文段之后，必须发出确认：ACK=1,seq=u+1,ack=w+1。 再经过2MSL(最长报文端寿命)后，本次TCP连接真正结束，通信双方完成了他们的告别 【客户端：“好的收到，连接2MSL后将关闭”】

    • 服务器代理

    • 报文：

      • 分类：请求报文 、 响应报文

      • 组成 = 开始行（区分请求报文和响应报文） + 首部行 + 实际主体

      • 请求报文的方法：

      方法（操作） 意义
      option 请求一些选项的信息
      get 读取URL所标志的信息
      post 给服务器添加信息
      head 读取URL所标志的信息首部
      put 在指明的URL下存储文档
      delete 删除UPL所标志的资源
      trace 用来进行回环测试的请求报文
      connect 用于代理服务器
      • post和get的区别
      区别 get post
      传输 明文传输，可见参数和值 不可见参数和值
      安全性 低 高
      传输数据 少量数据，IE下为255个字符 大量数据，值可配置
      提交原理 提交的数据间是独立的 将数据转化成XML格式提交
      灵活性 灵活，只要有页面跳转就可以传送数据 不灵活
