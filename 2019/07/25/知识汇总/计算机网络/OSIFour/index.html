运输层是整个网络体系中的关键层次之一，主要TCP,UDP的内容

1、传输层作用:

  • 用于向它上面的应用层特供通信服务

  • 提供进程间的逻辑通信（网络层提供主机间的逻辑通信）

2、TCP 与 UDP 协议

区别 TCP UDP
中文 传输控制协议 用户数据报协议
运输数据单元 TCP报文段 UDP数据报
连接 面向连接的服务 无连接
是否可靠 可靠的 不可靠的，尽最大努力交付的
传输 面向字节流 面向报文
通信 点<=>点即一对一 支持 一/多 <=> 一/多 通信
拥塞机制 全双工通信 无拥塞控制
首部开销 大 小

TCP

1、套接字（socket）：

  • 定义：TCP是端到端的连接，而TCP的端点即为套接字也叫插口;同时两个套接字也确定一条TCP连接

  • socket = IP地址:端口号 eg: 若IP地址为192.3.4.5，端口号为8080 ==> socket = 192.3.4.5:8080

2、可靠传输原理：

  • 停止等待协议

    • 无差错情况：A（发送分组）->B、A等待B确认、B（发送确认）->A、A收到确认、A下一轮发送……

    • 出现差错：A（发送分组）->B、A等待B确认、B接收发生错误的分组报文->B丢弃分组、A没收到确认、A超时重传……

    • 确认丢失：A（发送分组）->B、A等待B确认、B（发送确认）->A、A没收到确认、A超时重传、A等待B确认、B（发送确认）->A……

    • 确认迟到：A（发送分组）->B、A等待B确认、B（发送确认）->A、A没收到确认、A超时重传……A收到迟到的确认信息、A什么都不做……

  • 连续ARQ协议

    • 提升信道利用率、容易实现；但不能反映接收方已正确的分组信息

    • A（发送n个分组)->B,B（发送收到的最后一个分组m）->A,A收到确认，A（发送m+1开始的分组）->B

3、流量控制

  • 定义：让发送方的发送速率不要太快，以便于接收方来得及接收

  • 控制方法：

    • A->B、B（发送确认，同时发送自己空闲缓存区的大小）->A、A（调整发送的速率和大小）->B

    • 若A收到B 接收窗口大小 = 0 –> A停止发送数据、A开启定时器

    • 定时器间隔一定时间探测B的接收窗口大小，若仍为0，重新开启；否则A发送分组

    PS: 空闲缓存区的大小 = 接收窗口大小

4、拥塞控制

  • 定义：（资源需求量 > 资源可用量时即为拥塞）防止过多数据注入网络，造成网络中路由器或链路过载

  • 方法：

    PS:拥塞窗口（cwnd）为一个状态变量，大小取决于网络的拥塞程度，能够动态变化。发送方的发送窗口 = 拥塞窗口

    • 慢开始：由小到大增大拥塞窗口的数值,且每过一轮，cwnd = cwnd * 2 <== （由于在开始时不清楚网络的负载情况，所以从小到大增大发送窗口）

    • 拥塞避免:由小到大增大拥塞窗口的数值,且每经过一段时间，cwnd = cwnd + 1

    • 快重传： B自觉的定期发送到目前为止收到的分组，同时A根据B的确认判断分组的发送情况，若出现失误，立即重传错误分组

    • 快恢复：调整拥塞窗口的值，让 cwnd = cwnd / 2，并开始拥塞避免。此“快”相对于慢开始而言。因为慢开始是让cwnd = 1

5、运输连接管理

  • 三次握手：建立连接阶段

    • 第一次：客户端向服务器发送请求报文段，其首部中：SYN=1,seq=x，并进入SYN_SEND状态，等待服务器确认。（SYN：同步序列编号）【客户端：“服务器~ 我想要建立连接”】

    • 第二次：服务器收到并确认,若同意请求，则发送确认报文段，首部中SYN=1,ACK=1,ack=x+1,seq=y 【服务器：“可以”】

    • 第三次：客户端收到确认报文之后，通知上层应用进程连接已建立，并向服务器发出确认报文，其首部：ACK=1,ack=y+1。当服务器收到客户端的确认报文之后，也通知其上层应用进程连接已建立。 【客户端：“已告诉应用层连接建立了”】【服务器：“我也已经告诉应用层连接建立了”】

    • 四次挥手：连接释放阶段

      • 第一次：数据传输结束以后，客户端的应用进程发出连接释放报文段，并停止发送数据，其首部：FIN=1,seq=u 【客户端：“我的数据传完了”】

      • 第二次：服务器端收到连接释放报文段之后，发出确认报文，其首部：ack=u+1,seq=v。此时本次连接就进入了半关闭状态，客户端不再向服务器发送数据。而服务器端仍会继续发送 【服务器：“好的，那我关闭你向我传递数据的连接啦，不过我还没传完，客户端你先别关闭我向你传递数据的连接噢”】

      • 第三次：若服务器已经没有要向客户端发送的数据，其应用进程就通知服务器释放TCP连接。这个阶段服务器所发出的最后一个报文的首部应为：FIN=1,ACK=1,seq=w,ack=u+1 【服务器：“客户端我传完啦，你可以关闭连接了”】

      • 第四次：客户端收到连接释放报文段之后，必须发出确认：ACK=1,seq=u+1,ack=w+1。 再经过2MSL(最长报文端寿命)后，本次TCP连接真正结束，通信双方完成了他们的告别 【客户端：“好的收到，连接2MSL后将关闭”】
